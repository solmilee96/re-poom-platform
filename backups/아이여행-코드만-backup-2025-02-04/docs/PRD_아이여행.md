# 아이 여행 PRD (Product Requirements Document)

**서비스명**  
아이 여행 — '왜 이럴까?'를 '그래서 그랬구나'로 바꿔주는, 아이 기질 기반 행동 가이드 서비스

**문서 범위**  
핵심 가설, 문제 정의, 타겟 사용자, 핵심 기능만 기술합니다.

---

## 1. 문제 정의

- **양육자는 아이의 반복되는 행동(떼쓰기, 수면·식사 거부, 옷 입기 거부, 공격적 행동 등)을 “이상한 것”이나 “내 탓”으로 해석**하고, 그래서 죄책감·불안·무력감을 느낀다.
- **일반적인 양육 조언은 “아이마다 다르다”는 전제 없이 제시**되는 경우가 많아, 우리 아이에게 왜 안 맞는지, 무엇부터 바꿔야 할지 모호하다.
- **기질(temperament)에 대한 이해가 부족**해, 아이의 행동을 “버릇”이나 “고집”으로만 보기 쉽고, 그 결과 훈육·습관 형성에서 일관성이 떨어진다.

**정리:**  
“왜 우리 아이만 이럴까?”라는 질문에, **기질 기반의 이해와 맞춤 전략**이 없어서 양육자가 방향을 잡기 어렵고, 관계와 양육 효능감이 떨어진다.

---

## 2. 핵심 가설

- **가설 1:**  
  아이의 행동을 **기질(선천적 성향)** 로 설명해 주면, 양육자는 “문제”가 아니라 “이 아이의 특성”으로 받아들이고, **“그래서 그랬구나”** 로 인식 전환이 일어난다.
- **가설 2:**  
  **기질 유형별·상황별로 구체적인 대화법·루틴·환경 조정**을 제시하면, 일반론보다 **실제로 시도할 가능성과 지속 가능성**이 높아진다.
- **가설 3:**  
  **연구·이론(Thomas & Chess, Gottman, Dunn 등)을 인용한 설명**을 함께 주면, “그냥 조언”이 아니라 “근거 있는 정보”로 받아들여져 **신뢰와 이행률**이 올라간다.

**검증 방향:**  
서비스 이용 후 “이해도·적용 의지·양육 스트레스·만족도” 변화를 정성·정량적으로 확인한다.

---

## 3. 타겟 사용자

| 구분 | 설명 |
|------|------|
| **1차** | **영·유아(0~36개월 이상)를 양육하는 보호자** — 특히 떼쓰기, 수면, 식사, 옷 입기, 또래 관계 등에서 반복적인 어려움을 겪는 사람. |
| **2차** | 아이의 행동을 “이상하다”거나 “내가 못 키우는 건가”라고 느끼며 **정서적 부담**을 겪는 부모. |
| **공통 니즈** | “우리 아이만 이런 게 아닐까”에 대한 안심, **우리 아이에게 맞는** 구체적 방법, 부담이 크지 않은 접근(짧은 문항·바로 결과·필요 시 추가 상담). |

---

## 4. 핵심 기능

1. **아이 정보 입력**  
   - 아이 이름, 생년월일 입력.  
   - 이후 모든 결과·상담은 “이 아이” 기준으로 맞춤화.

2. **연령별 기질 문항 (9차원)**  
   - 만 나이(개월)에 따라 **0~11개월 / 12~23개월 / 24개월+** 중 해당 문항 세트 제공.  
   - **9개 차원:** 활동성, 규칙성, 접근성, 적응성, 반응 강도, 기분, 주의 산만성, 지속성, 감각 민감도.  
   - 문항 응답으로 **아이 기질 유형**(에너자이저, 신중파, 섬세파, 탐험가, 순둥이, 밸런서, 혼합형) 및 **양육자 유형** 도출.

3. **기질 결과지**  
   - 아이 기질 유형·특징·강점·민감한 상황을 **한눈에 보기** 쉽게 요약.  
   - “왜 그럴 수 있는지”를 기질 관점에서 짧게 설명.  
   - (선택) 결과지 저장·공유.

4. **유형별 맞춤 가이드**  
   - 기질 유형별 **실천 팁·대화 예시·루틴 제안** (연구·출처 인용).  
   - 상황별(떼쓰기, 잠, 밥, 옷 입기, 또래 관계 등) **짧은 가이드** 제공.

5. **추가 상담(채팅형 Q&A)**  
   - 사용자가 **지금 힘든 상황**(떼쓰기, 수면, 식사, 옷, 공격 행동 등)을 텍스트로 입력.  
   - **아이·양육자 유형 + 입력 내용**을 반영한 **규칙 기반 답변** (피드백/이어짐 질문/주제별 첫 질문/기본 답변 분기).  
   - 같은 아이에 대해 **이전 상담 이력(localStorage)** 을 활용해 맥락 유지.

6. **피드백 수집**  
   - 서비스에 대한 만족/아쉬움(**“아쉬워요”** 등) 수집.  
   - (현재 구현) Vercel API 경유 Slack 등으로 전달해 서비스 개선에 활용.
